# 需求缺口分析

> 日期: 2026-06-29
> 方法: 以用户需求为起点，对齐市面成熟产品（滴答清单/TickTick、时光序、鲨鱼记账、Notion），逐模块分析功能缺口。

---

## 一、分析基准

### 对标产品

| 产品 | 对标领域 | 核心优势 |
|------|----------|----------|
| **滴答清单 / TickTick** | 日历+待办+目标 | 事件分类、优先级标记、重复日程、今日概览 |
| **时光序** | 全能型日程+专注 | 今日仪表盘、多视图整合 |
| **鲨鱼记账 / 随手记** | 记账 | 图表分析、周期性记账、分类占比 |
| **Notion / 语雀** | 笔记 | 置顶、格式化、分类标签、搜索 |
| **Google Calendar** | 日历 | 拖拽调整时间（默认开启）、重复事件、颜色标签 |

---

## 二、缺口清单

### 🔴 高优先级（高频刚需 + 市面标配 + 用户确认）

#### 1. 全局搜索
- **对标**: 滴答清单、Notion、时光序
- **场景**: 用户想找某个笔记、某个日程、某个目标，目前只能逐个模块翻找
- **现状**: 仅记事本模块有搜索
- **目标**: Header 加搜索图标，跨模块搜索（事件标题、笔记标题+内容、目标标题）

#### 2. 重复日程
- **对标**: Google Calendar、滴答清单
- **场景**: "每天早上9点提醒"、"每周一开会"、"每月1号交房租"
- **现状**: 每次都需要手动创建同类型事件
- **目标**: 创建事件时增加重复选项（每天/每周/每月/工作日/自定义）

#### 3. 事件分类/颜色标签
- **对标**: Google Calendar、滴答清单、Fantastical
- **场景**: 一眼区分工作、生活、学习事件（工作日历中灰色工作事件、蓝色学习事件）
- **现状**: 所有事件外观完全相同
- **目标**: 6-8 色调色板，事件卡片左边框/背景色区分

#### 4. 待办优先级标记
- **对标**: Todoist（P1-P4）、滴答清单（高/中/低/无）
- **场景**: 10 个待办堆在一起，不知道先做哪个
- **现状**: 纯列表，无优先级区分
- **目标**: 高/中/低/无 四级标记，按优先级排序（可选）

#### 5. 记账消费图表
- **对标**: 鲨鱼记账、随手记
- **场景**: 记了一个月的账，想知道"餐饮花了多少"、"环比涨了还是跌了"
- **现状**: 纯列表展示，无统计可视图表
- **目标**: 饼图（分类占比）+ 趋势折线图（月度支出变化）

---

### 🟡 中优先级（有明显使用场景 + 市面常见）

#### 6. 周期性支出
- **对标**: 随手记
- **场景**: 房租、订阅费每月固定，不需要每次手动记
- **目标**: 支出可设置周期（每月/每周/每季），到期自动记录

#### 7. 笔记置顶
- **对标**: Notion、语雀
- **场景**: 重要笔记（如常用账号、近期项目大纲）需要永远在列表顶部
- **目标**: 笔记标记"📌 固定"，置顶分组永远在最上方

#### 8. 笔记简单格式化
- **对标**: 所有笔记软件（加粗、列表、标题）
- **场景**: 写笔记纯文本没有层级，"分段分点"无法区分
- **目标**: 支持 Markdown 基础语法或工具栏按钮（加粗、无序列表、有序列表）

#### 9. 今日仪表盘
- **对标**: 时光序、滴答清单首页
- **场景**: 每天打开 App 切日历→切待办→切目标才知道今天有什么事
- **目标**: 今日概览视图（今天的事件 + 待办 + 到期目标），可选作为默认首页

#### 10. 记账数据导出 CSV
- **对标**: 所有记账软件
- **场景**: 想把记账数据导入 Excel 做自定义分析
- **现状**: 只有 JSON 全量导出，非结构化
- **目标**: 记账模块支持导出 CSV（日期、金额、分类、备注）

#### 11. 拖拽调整时间默认开启
- **对标**: Google Calendar
- **场景**: 用户想快速拖拽事件改时间
- **现状**: 藏在设置里默认关闭，大部分用户不知道
- **目标**: 默认开启，设置里可关闭

#### 12. 待办截止日 vs 提醒日分离
- **对标**: 滴答清单
- **场景**: "周五交报告，但周三提醒我开始写"
- **现状**: 截止日和提醒日绑在一起（都是事件日期）
- **目标**: 待办增加独立的提醒时间字段

---

### 🟢 低优先级（锦上添花）

#### 13. 事件备注/详情
- **对标**: Google Calendar（支持富文本描述）
- **场景**: 想记录会议纪要、地址等附加信息
- **目标**: 事件编辑表单增加备注/描述字段

#### 14. 目标模板
- **对标**: 滴答清单（"读书计划""健身计划"模板）
- **场景**: 每次创建目标从零搭建
- **目标**: 预设目标模板（读书 N 本、健身 N 次、学习 N 小时）

## 三、设计决策记录

以下需求在讨论中已被明确排除：

| 需求 | 排除原因 |
|------|----------|
| AI 解析预览 | 增加操作步数，用户希望直接创建不额外确认 |
| 删除操作确认 | 已有垃圾桶可还原，弹窗打断体验 |
| 手势操作视觉优化 | 改动大收益小 |
| 筛选空状态提示 | 已有空状态设计 |
| 目标进度可视化 | 不可靠，数据基础不足 |
| 快捷键系统 | 学习成本高 |
| 通知系统（Web/桌面） | QQ 通知已满足需求 |
| 深色模式 | 当前优先级低 |
| Onboarding 引导 | 改动大，用户学习意愿强 |
| 数据统计看板 | 与今日仪表盘有重叠，低保 |
