# AI 交互体验改进方案

## 当前问题

| 问题 | 具体表现 | 用户感受 |
|------|---------|---------|
| 黑盒执行 | 发消息 → 转圈 → 结果替换，中间无信息 | 不知道 AI 在做什么，缺乏掌控感 |
| 无流式输出 | 必须等完整响应才显示 | 等待焦虑，无法提前阅读 |
| 全量替换 | `/a` 指令直接替换整个笔记内容 | 不知道改了哪里，改坏了要手动撤销 |
| 无对比 | 改前改后没有对照 | 没有决定感，无法选择 |
| 无建议列表 | AI 只给一个答案，不给选项 | 不能选最合适的版本 |
| 无思考过程 | "思考中..." 不知道想什么 | 黑箱感 |

## 目标

参考 AI 编程工具（Cursor、Copilot、Claude Code）的设计，让笔记 AI 交互变成 **可见、可控、可选择** 的体验。

---

## 方案设计

### 1. 流式输出（Streaming）

**当前**：`POST /api/llm/chat-agent` → 等完整响应 → 一次性显示

**改进**：后端支持 SSE（Server-Sent Events）或前端模拟分块显示

**前端模拟流式**（后端不改）：
```
用户发送消息 → AI 返回完整内容
  → 前端用 setInterval 逐字/逐段显示在界面上
  → 模拟打字效果（每 30-50ms 显示一个字符）
  → 用户可提前阅读，减少等待感
```

**后端真流式**（后期）：
```
POST /api/llm/chat-agent/stream
  → SSE 响应，每个 chunk 推送到前端
  → 前端实时追加到界面
  → 真正的逐字显示
```

**推荐先做前端模拟流式**，改动小，效果明显。

---

### 2. 思考过程展示

**当前**：只显示「思考中...」

**改进**：AI 返回的内容可以分两部分：

```
思考过程（可选展示）：
  - 分析用户需求
  - 检查笔记内容
  - 考虑要修改的部分

最终结果：
  - 润色后的完整内容
```

**实现方式**：
- 后端 prompt 中要求 AI 用 `[思考]...[/思考]` 标记思考过程
- 前端解析这个标记，分别显示「思考」和「结果」
- 思考过程用灰色/斜体展示，可折叠

**示例显示**：
```
┌─────────────────────────────────────┐
│ 💭 思考过程（点击展开）              │
│ 用户要求润色这段文字，使其更正式。    │
│ 原文是口语化表达，需要改为书面语。    │
│ 同时保留了关键信息：LinuxDo 通过审核 │
├─────────────────────────────────────┤
│ ✅ 润色结果                         │
│ 今日我的LinuxDo账号已通过审核...      │
│                    [接受] [拒绝]     │
└─────────────────────────────────────┘
```

### 3. 差异对比（Diff）

**当前**：直接替换全文，用户不知道改了哪里

**改进**：显示修改前后的差异对比

```
┌─ 原文 ──────────────────────────────┐
│ 今天我的linuxdo通过了，我得研究一下    │
├─ 修改后 ────────────────────────────┤
│ 今日我的LinuxDo账号已通过审核，        │
│ 现需对其进行研究。                    │
├─────────────────────────────────────┤
│ 变更：                              │
│ 🟢 +"账号"  🟢 +"已通过审核"        │
│ 🔴 -"我得"   🔴 -"一下"             │
│ 🟡 修改 "linuxdo" → "LinuxDo"       │
│                    [接受] [拒绝]     │
└─────────────────────────────────────┘
```

**简化实现**（不用真正的 diff 算法）：
- 在 `/a` 指令中，AI 返回时带上修改说明
- 前端把修改说明展示为变更列表
- 或者用简单的行级对比（前后文本按行分割对比）

---

### 4. 接受/拒绝机制

**当前**：AI 结果直接应用，不可选

**改进**：每次 AI 修改都进入「待确认」状态

```
AI 完成修改 → 显示 diff 预览
              → 用户选「接受」→ 应用到笔记
              → 用户选「拒绝」→ 保持原样
              → 用户选「换一个」→ AI 再生成一个版本
```

**UI 设计**：

```
┌─────────────────────────────────────┐
│ 🤖 润色建议                         │
│                                     │
│ 今日我的LinuxDo账号已通过审核...      │
│                                     │
│ [✓ 接受] [✗ 拒绝] [↻ 换一个]        │
└─────────────────────────────────────┘
```

**与 Undo 的关系**：
- 接受 → 保存到笔记 + 进入 undo 栈（已有）
- 拒绝 → 不保存，内容不变
- 换一个 → 重新请求 AI，覆盖当前建议

### 5. 多版本建议

**当前**：AI 只给一个答案

**改进**：AI 给出多个版本，用户选择

```
用户：润色这段话

AI 返回：
┌─ 版本 A ───────────────────────────┐
│ 今日我的LinuxDo账号已通过审核...      │
├─ 版本 B ───────────────────────────┤
│ LinuxDo 账号已获批准，值得深入探索    │
├─ 版本 C ───────────────────────────┤
│ 今日获悉LinuxDo账号已通过审核...     │
├─────────────────────────────────────┤
│ [使用 A]  [使用 B]  [使用 C]         │
└─────────────────────────────────────┘
```

**实现**：prompt 中要求 AI 返回 2-3 个版本，用分隔符区分。

### 6. 加载动画

**当前**：「思考中...」一行灰字

**改进**：多层级的加载反馈

```
发送后立即：
  ┌────────────────────────────┐
  │ 🤖 AI 正在分析笔记...      │
  │ ═══════════════════░═══    │ ← 进度条（模拟）
  │ 已读取 1 条笔记 · 正在处理  │
  └────────────────────────────┘

500ms 后：
  ┌────────────────────────────┐
  │ 💭 思考：用户要求润色       │
  │ 正在分析原文结构...         │

1.5s 后：
  ┌────────────────────────────┐
  │ 📝 正在生成润色版本...      │

完成后：
  ┌────────────────────────────┐
  │ ✅ 完成，共修改 3 处        │
  │ [接受] [拒绝] [换一个]      │
  └────────────────────────────┘
```

**实现**：用 `setTimeout` 分阶段更新提示文字，模拟进度。

### 7. 内联 Ghost Text（类似 Copilot）

**当前**：AI 通过 `/a` 或抽屉交互

**改进**：在笔记编辑器内直接显示 AI 建议的灰色占位文本

```
用户在 contenteditable 中打字：
  今天下午我[要去开会...]
                     ↑ 灰色虚线文字
                    按 Tab 接受建议
```

**实现方式**：
- 监听用户的输入停顿（>1.5s 无输入）
- 自动将当前段落发送给 AI 做续写/补全
- 返回的结果以 `<span class="ai-ghost">` 插入到光标后
- 按 Tab → ghost text 转为正式文本
- 继续打字 → ghost text 消失

**复杂度高，建议后期再做**。

## 实施优先级

| 优先级 | 功能 | 改动量 | 效果 |
|--------|------|--------|------|
| P0 | 流式显示（至少前端模拟打字效果） | 小 | 消除黑盒感 |
| P0 | 分阶段加载动画 | 小 | 减少等待焦虑 |
| P1 | 接受/拒绝机制 | 中 | 增加掌控感 |
| P1 | Diff 对比（简单行级或变更列表） | 中 | 知道改了哪里 |
| P2 | 思考过程展示 | 中 | 增加透明度 |
| P2 | 多版本建议 | 中 | 增加选择权 |
| P3 | 后端真流式 SSE | 大 | 最佳体验 |
| P3 | Ghost text 内联建议 | 大 | 类 Copilot 体验 |

---

## 推荐先做（P0 + 部分 P1）

**第一轮（P0）**：不做任何后端改动，只改前端
- `note-editor.js`：`/a` 指令增加分阶段动画 + 模拟打字效果 + 接受/拒绝按钮
- `note-ai.js`：AI 抽屉增加打字效果
- `style.css`：动画样式

**第二轮（P1）**：
- 添加 Diff 预览（前后对比）
- 添加「换一个」选项
- 修改 prompt 让 AI 返回结构化内容（思考+结果+变更列表）

**第三轮（P2+）**：
- 多版本
- SSE 流式
- Ghost text

## 实现细节（P0）

### 模拟打字效果

```javascript
async function _simulateTyping(element, text, speed = 30) {
    let i = 0;
    return new Promise(resolve => {
        const interval = setInterval(() => {
            if (i >= text.length) {
                clearInterval(interval);
                resolve();
                return;
            }
            // 每步追加 1-3 个字符（模拟真实打字节奏）
            const chunkSize = Math.min(1 + Math.floor(Math.random() * 2), text.length - i);
            element.textContent += text.substring(i, i + chunkSize);
            i += chunkSize;
            // 滚动到底部
            element.parentElement.scrollTop = element.parentElement.scrollHeight;
        }, speed);
    });
}
```
